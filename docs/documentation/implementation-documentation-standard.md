# Implementation Documentation Standard

Every material implementation PR must include synchronized documentation updates.

## Required Updates by Change Type

- API contract changes:
  - update endpoint docs
  - update request/response examples
- Tooling and CI changes:
  - update README command sections
  - update local runbook and contribution guidance
- Architecture/boundary changes:
  - update relevant RFC/ADR and ownership mapping

## Definition of Done

- Code merged with matching docs in same PR
- PR checklist docs-sync items completed
- Reviewer confirms docs reflect effective behavior
